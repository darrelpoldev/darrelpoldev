# Review Code

- Purpose: review the code written by /start-coding
- Check the code fulfills the ticket's acceptance criteria
- Check the code follows the coding style and standards — review against `references/code-standards.md`
- Catch issues/bugs in the implementation
- Report findings ordered by severity; distinguish blockers from nitpicks
- After the review, suggest /observe-code and /secure-code
