# Plan With Me

- Purpose: interview the user until the requirements are actually understood — thorough, but never overwhelming
- Gather requirements only — the WHAT and WHY. Do NOT design the solution or decide how to build it; that's /techplan-with-me's job
- Input can be a rough idea, a /talk-to-me dump, or an existing ticket — ticket work ALWAYS passes through here before coding
- Interview cadence: ONE question at a time, 2 options where possible
- Dig into goals, constraints, edge cases, and unknowns — depth comes from many small questions, not big heavy ones
- Capture the details of the specific feature/work — not a generic template
- Write the result to a file: a spec, plan, or RFC — whichever fits the work
- Keep iterating until the user and agent explicitly agree they're on the same page
- When you have enough information, suggest /itemize-work or /techplan-with-me — the user decides; never run them unprompted
- This is the ONLY path into /techplan-with-me — no technical plan without a plan, no code without a technical plan
